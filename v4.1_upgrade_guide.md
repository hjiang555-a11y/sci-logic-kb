# SCHEMA v4.1 论文升级指南

## 概述

本指南用于将sci-logic-kb知识库中的论文从v3.0/v3.1/v3.2升级到v4.1规范。v4.1要求每篇论文包含完整的推理链条：principles（原理）、methods（方法）、metrics（指标）和relations（关系）。

## 升级目标

- **技术论文**：必须包含1-2个principles、1-2个methods、2-4个metrics和至少3个relations
- **综述论文**：必须包含1-2个principles，methods和metrics可为空（或添加理论参数），至少2-3个relations
- **跨文件引用**：尽可能使用已有实体（ent.*）、方法（meth.*）、原理（pri.*）、指标（met.*）
- **质量优先**：每个字段都应基于论文实际内容，避免空洞描述

## 字段要求

### 1. Principles（原理）

- **ID格式**：`pri.[描述性名称]_[作者年份缩写]`，如 `pri.low_noise_pd_design_g16`
- **内容要求**：核心科学原理、设计理念、理论创新
- **技术论文**：提取实验/设计原理，如"高功率低噪声光电探测器设计"
- **综述论文**：提取总结性原理，如"高斯光束谐振腔理论"
- **证据级别**：`theoretical`（理论）、`experimental`（实验）、`observed`（观察）、`inferred`（推断）

### 2. Methods（方法）

- **ID格式**：`meth.[描述性名称]_[作者年份缩写]`，如 `meth.low_noise_pd_circuit_g16`
- **内容要求**：具体实验方法、工程实现、分析技术
- **技术论文**：必须包含，描述如何实现原理
- **综述论文**：可为空，或添加分析方法
- **输入/输出**：明确方法输入和输出

### 3. Metrics（指标）

- **ID格式**：`met.[指标名称]_[作者年份缩写]`，如 `met.noise_reduction_g16`
- **内容要求**：关键性能指标，包括数值、单位、测量条件
- **技术论文**：2-4个具体指标，最好有数值
- **综述论文**：可为空，或添加理论参数
- **字段**：`quantity`（物理量）、`unit`（单位）、`demonstrated_value`（演示值）、`conditions`（条件）、`measurement_method`（测量方法）、`significance`（重要性）

### 4. Relations（关系）

- **数量要求**：至少3个（技术论文），至少2个（综述论文）
- **关系类型**：
  - `ENABLED-BY`：实体由原理实现
  - `IMPLEMENTS`：方法实现原理
  - `CHARACTERIZES`：指标表征方法/原理
  - `OPERATIONALIZED-AS`：实体操作方法
  - `BOUNDED-BY`：受限于原理/指标
  - `COMPETES-WITH`：与...竞争（慎用）
  - `PART-OF`：部分关系
- **置信度**：`established`（确立）、`likely`（可能）、`speculative`（推测）

## 区别处理：综述论文 vs 技术论文

### 技术论文（source_type: technical_paper）
```
原则：必须完整
方法：必须完整（1-2个）
指标：必须完整（2-4个）
关系：至少3个
```

### 综述论文（source_type: review_paper）
```
原则：必须完整（1-2个总结性原理）
方法：可为空（或1个分析方法）
指标：可为空（或理论参数）
关系：至少2-3个
```

## 升级流程

### 步骤1：分析现有内容
1. 读取YAML文件，检查现有字段
2. 识别缺失字段
3. 阅读论文摘要/笔记，理解核心贡献

### 步骤2：补充缺失字段
按照以下优先级：
1. 补充principles（如有缺失）
2. 补充methods（技术论文必须）
3. 补充metrics（技术论文必须）
4. 扩展relations至最少数量

### 步骤3：创建逻辑链条
- 原理 → 实现 → 方法
- 方法 → 表征 → 指标
- 实体 → 启用 → 原理
- 跨文件引用已有元素

### 步骤4：更新SCHEMA.md
修改论文状态从"✅ 符合v3.2（首次提取）"等变为"✅ v4.1（补充principles/methods/metrics/relations推理链条）"

## 示例模板

### 模板1：技术论文（完整升级）
```yaml
principles:
  - id: pri.[名称]_[年份缩写]
    name: "原理名称"
    statement: > 
      原理描述...
    domain: "领域"
    subdomain: "子领域"
    foundation: engineering/theoretical
    evidence: experimental/observed

methods:
  - id: meth.[名称]_[年份缩写]
    name: "方法名称"
    description: >
      方法详细描述...
    inputs: ["输入1", "输入2"]
    outputs: ["输出1", "输出2"]

metrics:
  - id: met.[指标]_[年份缩写]
    name: "指标名称"
    quantity: "物理量"
    unit: "单位"
    demonstrated_value: "数值"
    conditions: "条件"
    measurement_method: "测量方法"
    significance: "重要性"
```

### 模板2：综述论文
```yaml
principles:
  - id: pri.[总结性原理]_[年份缩写]
    name: "总结性原理"
    statement: >
      理论总结...
    domain: "理论领域"
    foundation: theoretical
    evidence: observed/inferred

methods: []  # 可为空

metrics:
  # 可选：理论参数
  - id: met.[理论参数]_[年份缩写]
    name: "理论参数"
    quantity: "参数"
    unit: "单位"
    demonstrated_value: "理论公式"
    conditions: "理论条件"
    measurement_method: "理论计算"
    significance: "理论重要性"
```

## 质量检查

升级后应满足：
1. 语法正确：YAML格式无错误
2. 内容相关：字段基于论文实际内容
3. 逻辑一致：relations连接合理
4. 格式规范：ID格式正确，字段完整
5. 跨文件引用：优先使用已有元素

## 批量处理策略

### 批次划分
1. **测试批次**：5-10篇简单任务（缺失≤2字段）
2. **主要批次**：30-40篇中等任务（分3-4批）
3. **复杂批次**：4-8篇复杂任务（单独处理）

### 时间分配
- 每批次目标：30-60分钟处理时间
- 简单任务：每篇5-8分钟
- 中等任务：每篇10-15分钟
- 复杂任务：每篇20-30分钟

### 验证脚本
使用`validate_v4.1.py`检查升级质量：
```bash
python validate_v4.1.py --file paper.yaml
```

## 常见问题

### Q: 论文中没有明确数值指标怎么办？
A: 使用定性描述，如"显著改善"、"优于基准"，或添加理论参数。

### Q: 如何确定principles？
A: 问：论文的核心科学贡献是什么？设计理念是什么？理论创新是什么？

### Q: 跨文件引用找不到合适元素？
A: 创建新元素，但检查是否已有相似元素可复用。

### Q: relations不够3个怎么办？
A: 创建更多逻辑连接：原理-方法、方法-指标、实体-原理、指标-原理等。
