## 二、重新梳理原则

### 2.1 Schema一致性原则
- 所有论文必须遵循SCHEMA.md v4.1（当前最新版本）
- 统一使用`# Schema版本：v4.1`头部注释
- 确保所有字段定义与Schema一致

### 2.2 推理链条完整性原则
每个技术限制（`BOUNDED-BY`关系）应尽可能配有以下元素：
1. **问题描述**：通过关系上下文和`note`字段清晰描述技术限制
2. **解决方案追踪**：通过`breakthrough_paths`字段记录突破路径
3. **结果验证**：通过`status`、`verification_status`、`temporal_role`字段记录证据状态
4. **证据溯源**：每个重要论断必须有`source`字段引用原文

### 2.3 关键技术分支映射原则
每篇论文应清晰归属到以下超稳激光技术分支之一：
1. **FP参考腔系统**（Fabry-Pérot Reference Cavity Systems）
2. **光纤延迟线/干涉仪参考**（Fiber Delay Line/Interferometer References）
3. **光谱烧孔(SHB)频率参考**（Spectral Hole Burning Frequency References）

对于方法类论文，应归属到：
4. **误差探测方法**（PDH、Tilt Locking等）
5. **稳频策略**（多级级联、多腔平均等）

## 四、具体操作指南

### 4.1 更新论文YAML文件的步骤
对于每篇论文YAML文件，执行以下操作：

**步骤1：更新头部信息**
```yaml
# {Author} {Year} — {论文标题}
# 提取者：{AI模型}（AI草稿，待专家确认）
# 提取日期：{当前日期}
# Schema版本：v4.1  # 更新为v4.1
```

**步骤2：检查并完善`breakthrough_paths`字段**
对于每个`BOUNDED-BY`关系，确保包含`breakthrough_paths`字段：
```yaml
breakthrough_paths:
  - direction: pri.some_principle  # 必须引用pri.*或meth.*
    expected_gain: "预期性能提升描述"
    status: proposed  # proposed | demonstrated | refuted
    source: {zotero_key: "KEY", claim: "原文论断"}
    note: "可选的补充说明"
```

**步骤3：补充推理相关字段**
在原理节点中：
- 检查并补充`open_questions`字段（记录未解问题）
- 检查并补充`contested_claims`字段（记录有争议的论断）
- 检查并补充`preconditions`和`invalidated_when`字段

**步骤4：验证证据溯源**
- 确保每个重要论断有`source`字段
- 确保跨论文引用在`note`中说明
- 确保指标值有`conditions`字段说明测量条件

**步骤5：技术分支归属**
在`meta.note`或相关实体节点的`note`中说明论文的技术分支归属。

### 4.2 AI提取指导要点
当使用GitHub Copilot或Claude进行重新梳理时，注意：

1. **优先提取推理元素**：
   - 先识别技术限制（`BOUNDED-BY`）
   - 再提取突破路径（`breakthrough_paths`）
   - 最后记录验证状态（`status`, `verification_status`）

2. **保持一致性**：
   - 复用已有节点定义，不重复定义
   - 遵循全局唯一ID原则
   - 使用标准化的字段格式

3. **注重证据质量**：
   - 直接引用原文论断，不进行解释或总结
   - 区分实验测量（`observed`）和理论推导（`calculated`）
   - 记录论文在知识演进中的角色（`temporal_role`）
