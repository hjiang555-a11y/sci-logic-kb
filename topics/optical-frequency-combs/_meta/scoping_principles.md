# Optical Frequency Combs — 专题内部评判原则

> **版本**：v2（2026-04-21）
> **范围**：本文件仅在 `topics/optical-frequency-combs/` 范围内有效，**不**向全局 knowledge base 扩展。
> **与全局规则的关系**：全局规则在 [`SCHEMA.md`](../../../SCHEMA.md) 与 [`docs/CONTRIBUTION_TIER_RULES.md`](../../../docs/CONTRIBUTION_TIER_RULES.md) 中定义；本文件补充**只针对光学频率梳专题**的评判偏好。冲突时，本文件的专题偏好**优先**用于本专题内的档位判定；跨专题讨论仍以全局规则为准。
>
> **v2 建立说明**：v1 中各子域主线指标标注"🟡 待专家确认"；随着 30 篇 OFC 论文批量摄入，专家明确指出 **光频梳不同于超稳激光，不存在单一主线指标**——不同子方向的关键指标天然不同（例如光谱展宽强调绝对 PSD 与转换效率，高重频源强调 f_rep 与相噪，应用侧各自分散）。v2 将各子域主线指标从"候选/待确认"升级为正式定义，并给出每条主线的档位判据。新子域仍需积累 ≥ 3 篇再独立细化。

## 1. 专题结构与主线指标（Multi-Track Principle）

### 1.1 原则

> **光学频率梳专题不适合单一主线指标；按应用子域分别定义。**

动机：
- 频率综合/计量子域：关注的是梳齿到参考激光的**相位噪声传递**和**频率不确定度**
- 双梳光谱子域：关注的是**双梳互相干时间**（mutual coherence time）和**光谱覆盖宽度**
- 天文光梳子域：关注的是**梳齿均匀性**（comb line uniformity）和**长期稳定度**
- 微腔梳子域（DKS）：关注的是**泵浦效率**、**相位噪声**和**集成化 SWaP**

强制单一主线会导致大量非最优分类（尤其对于跨子域的综述/应用论文）。

### 1.2 子域主线指标（v2 正式定义）

光频梳 Multi-Track 主线指标体系——下表每一行都是一条**独立主线**，档位在该主线上独立判定。同一篇论文可同时刷新多条主线。

| 子域 | 主线指标（v2 正式） | 关键二级指标 | 典型代表文件 |
|------|------------------|------------|-------------|
| **A1-Rep 高重频梳技术** | f_rep（基频重复频率，非谐波放大）+ 自参考可行性 | fceo SNR（@RBW）、单齿功率、八度谱覆盖 | bartels2009（10 GHz）、ma2018（750 MHz）、wang2014（500 MHz） |
| **A1-Noise 低相噪/低时延抖动梳技术** | f_CEO 积分相位噪声（rad, 标注积分区间）或拍频积分相噪 | 积分时延抖动（as, 标注区间）、in-loop σ_y(1 s) | kuse2015/2016、li2017、ma2018 |
| **A1-Robust 工程化/全保偏/紧凑化梳** | 连续稳定时长 + SWaP（足迹/功耗） | 单驱动器带宽、环境适应范围、H-maser 比对残差 | cai2020、zhang2015、kuse2016 |
| **A2-DKS 微腔梳** | 相位噪声 @ offset + **泵浦-梳光转换效率** + 集成度 SWaP | f_rep、带宽、光谱展宽后带宽 | （已有）papp2013、stern2018 |
| **A3-Astro 天文光梳** | 梳齿均匀性（dB，经模滤波后）+ 长期稳定度（hours-scale） | 波长覆盖、单齿重复性 | NDSVHPF5 批次 3 |
| **B-Spec 光谱展宽/波长扩展** | **转移波段的绝对光谱 PSD**（W/Hz 或 dBm/Hz）+ **泵浦-信号转换效率** | 覆盖波段、相干传递保真度、σ_y 传递残差 | L88UAAEQ（XUV）、NDSVHPF5（UV–blue-green） |
| **B-FreqSyn 频率综合/计量链路** | σ_y(τ=1 s) 的光-光 / 光-微波传递残差 + 分发距离 | 梳齿-CW 拍频 SNR、多支路相位同步、相干链路长度 | P7A5Z647（10⁻¹⁸/3×）、U5C8AJYM（双支路）、SZL4V3W8（8 支路） |
| **B-DCS 双梳光谱** | 互相干时间（mutual coherence time）+ 光谱覆盖带宽 | 分辨率 × 带宽积、采集速率（高 f_rep 的速度优势） | PBGQXUED、NNLDMCDD、TFUBQQB8 |
| **B-MIR 中红外光谱** | 探测灵敏度（cm⁻¹/√Hz）+ 覆盖波段（μm） | 单模功率、分子指纹交叉灵敏度 | TFUBQQB8、NNLDMCDD |

> ⚠ **关键原则**（用户明确指示）：
> - 不要把所有论文都往"σ_y"这一条主线上塞（那是超稳激光的单主线做法）。
> - **光谱展宽类**论文档位判据优先看**转移后绝对 PSD** 与**转换效率**，不是看线宽或覆盖带宽本身。
> - **重复频率类**论文（如 10 GHz 以上）档位判据优先看**基频 f_rep + 是否 self-referenced**，不是看 σ_y。
> - **应用类**论文各自按 B-* 主线独立判档；综述/路线图归 framework。

### 1.3 子域内 `best_demonstration` 维护规则

每条主线独立维护"最佳历史纪录"——在该子域的代表节点（通常是 met.* 节点的 `historical_landmarks.best_demonstration`）中记录当前纪录持有者。若新论文刷新其中**任何一条**主线的纪录，即可按 `breakthrough` 档归类（不要求同时刷新所有主线）。

### 1.3 框架型论文（framework）的特殊处理

光学频率梳专题包含大量综述/路线图型论文（如 `picque2019.yaml`、`kippenberg2018.yaml`、`giunta2019.yaml`）。这类论文：

- 使用 `meta.contribution_type: framework`
- 主要职责是定义 Level 0/1 顶层实体与 tier: meta/domain 原理
- 不负责具体实验的 Level 2 参数实例
- **不要求**补 `breakthrough_paths`

## 2. 档位判据（v2 子域主线已正式定义）

在 v2 各子域主线指标基础上采用以下判据（优先级自上而下）：

| 信号 | 对档位的影响 |
|------|------------|
| **首次演示某新型梳技术原理**（如首篇 DKS、首篇双梳光谱、首篇 XUV 梳） | ✅ `breakthrough` |
| **刷新子域主线指标纪录**（按 §1.2 表格的主线指标判断） | ✅ `breakthrough` |
| **提出可复用的新 `pri.*` 原理**（不只是从已有原理派生） | ✅ `breakthrough` |
| 在已有梳技术上换材料/平台/波长复现，参数未破纪录 | ⚠ `evidence` |
| 综述 / 路线图 / 教科书 | `framework` |
| 其余（新数据点、工程改进、参数扫描） | `evidence`（默认） |

> 📌 **多主线刷新无需叠加**：刷新任一条子域主线即可升 breakthrough。不要求同时刷新多条主线。
> 📌 **跨子域论文**：若论文同时贡献两个子域（例如"高 f_rep + 低相噪"），可在 meta.note 中声明涉及的多条主线，但档位判定取**任意一条主线刷新纪录**即升 breakthrough。

## 3. 与超稳激光专题的接口

光学频率梳专题通过以下接口与超稳激光专题耦合：

- `CONDITIONED-BY ent.fp_cavity_system`：梳的相位噪声下限受询问激光稳定度约束
- `CONDITIONED-BY met.fractional_freq_instability_*`：梳齿频率不确定度受参考激光 σ_y 约束

处理跨专题边界论文时：
- 若主贡献在**超稳激光稳定度**而非梳技术 → 归 `ultrastable-laser` 专题
- 若主贡献在**梳技术/梳应用** → 归 `optical-frequency-combs` 专题

## 4. 遗留开放问题（v2 后保留）

> v2 已落实子域主线指标定义；以下问题为更深层的方法学讨论，不阻塞档位判定：

1. **B-FreqSyn 子域**：σ_y(τ) 是否应作为该子域的**统一化表征量**（类似超稳激光专题对 σ_y 的地位）？目前 v2 采用"σ_y 的光-光/光-微波传递残差"作为主线，但跨论文比较时 τ 基准不一致。
2. **B-DCS 子域**：互相干时间与"分辨率 × 带宽积"二者是否应合并为单一复合指标？目前 v2 列双指标，主线判档取任一刷新。
3. **A2-DKS 子域**：芯片集成度 SWaP 如何定量？目前以"集成度"定性描述——是否需要引入标准化 SWaP 评分？
4. **跨主线破纪录**：若论文同时刷新多条主线，是否应在 `meta.breakthrough_tracks` 中显式列出（避免 reviewer 遗漏）？当前仅在 note 中声明。
